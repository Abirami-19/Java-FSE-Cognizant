# Hands-on: Create Authentication Service that Returns JWT

## Objective

To understand the JWT authentication process and create an authentication service that generates a JSON Web Token (JWT) after validating user credentials.

## Description

JWT (JSON Web Token) is used to securely authenticate users in a RESTful application. The client sends the username and password to the authentication service using Basic Authentication. After successful validation, the server generates a JWT and returns it to the client. The client then includes this token in the Authorization header for subsequent requests to access secured REST APIs.

## Authentication Flow

1. The client sends the username and password to the `/authenticate` endpoint.
2. The authentication service reads the Authorization header.
3. The username and password are decoded from the Basic Authentication header.
4. The credentials are validated.
5. A JWT is generated for the authenticated user.
6. The generated JWT is returned to the client.
7. The client uses the JWT in the Authorization header (`Bearer <token>`) while accessing protected REST services.

## Sample Request


curl -s -u user:pwd http://localhost:8090/authenticate
